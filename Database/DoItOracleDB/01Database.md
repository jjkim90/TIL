## 1. 데이터와 데이터베이스, DBMS

- 데이터베이스는 데이터와 베이스의 합성어
- DBMS는 Database Management System의 약자로 '데이터베이스 관리 시스템'을 의미함.

- 오라클 - 유료 / 마리아DB - 무료



### 1.2. 데이터와 정보

- 데이터 : 어떤 필요에 의해 수집했지만 아직 특정 목적을 위해 평가하거나 정제하지 않은 값이나 사실 또는 자료 자체. 가공되지 않음.
- 정보 : 수집한 데이터를 어떠한 목적을 위해 분석하거나 가공하여 가치를 추가하거나 새로운 의미를 이끌어 낼 수 있는 결과. 가공됨.

- 효율적인 데이터 관리를 위한 조건
  - 데이터를 통합하여 관리
  - 일관된 방법으로 관리
  - 데이터 누락 및 중복 제거
  - 여러 사용자(응용 프로그램 포함)가 공동으로 실시간 사용 가능

- 데이터베이스 : 위의 조건을 만족하면서 특정 목적을 위해 여러 사람이 공유하여 사용할 수 있으며, 효율적인 관리와 검색을 위해 구조화한 데이터 집합.



### 1.2. 파일 시스템과 DBMS

- 데이터베이스 개념이 등장하기 전 주로 파일 시스템 방식을 사용하여 데이터를 관리하였음.



#### 파일 시스템을 통한 데이터 관리

- 파일 시스템을 단층 파일 구조(flat file structure)라고 함.
- 파일 시스템은 서로 다른 여러 응용 프로그램이 제공하는 기능에 맞게 필요한 데이터를 각각 저장하고 관리.
- 파일 시스템을 통한 데이터 관리의 문제점
  - 데이터 중복
  - 응용 프로그램이 개별 데이터를 직접 관리
  - 응용 프로그램이 데이터를 쓰는 방식이 각각 다름
  - 데이터가 특정 응용 프로그램에 종속되어 있으므로 응용 프로그램을 변경하면 기존 데이터를 사용할 수 없음.

- 여러 응용 프로그램이 사용할 데이터를 한곳에서 관리하기 위해 데이터베이스 활용.



#### DBMS를 통한 데이터 관리

- 데이터베이스 관리 시스템 : 데이터 조작과 관리를 극대화한 시스템 소프트웨어.
- 실무에서는 데이터베이스와 데이터베이스 관리 시스템을 따로 구별하지 않고 '디비(DB)' 또는 '데이터베이스'라고 부름.
- 데이터베이스를 통한 데이터 관리 : 여러 목적으로 사용할 데이터의 접근, 관리 등의 업무를 DBMS가 전담하는 방식.
- 작업 영역을 분리하여 응용 프로그램의 서비스 제공과 데이터 관련 작업 효율을 높임.
- 여러 응용 프로그램이 하나의 통합된 데이터를 같은 방식으로 사용, 관리할 수 있으므로 데이터 누락이나 중복을 방지할 수 있음.

- DBMS를 통한 데이터 관리의 장점
  - 하나의 소프트웨어가 데이터를 관리하므로 데이터 중복을 피할 수 있음.
  - 여러 응용 프로그램이 하나의 DBMS를 통해 데이터를 사용하므로 데이터를 동시에 공유할 수 있음.
  - 하나의 DBMS를 통해 데이터를 관리하기 때문에 각각의 으용 프로그램이 데이터를 관리하는 방식이 통합됨.
  - 응용 프로그램과는 별도로 데이터가 DBMS에 의해 관리, 보관되기 때문에 응용 프로그램의 업데이터 또는 변경과 관계없이 데이터를 사용할 수 있음.



## 2. 데이터 모델



### 2.1. 계층형 데이터 모델과 네트워크형 데이터 모델



#### 계층형 데이터 모델

- 나뭇가지 형태의 트리 구조를 활용하여 데이터 관련성을 계층별로 나누어 부모와 자식 같은 관계를 정의하고 데이터를 관리.



#### 네트워크형 데이터 모델

- 다대다. 자식 개체가 여러 부모 개체를 가질 수 있음.



### 2.2. 객체 지향형 데이터 모델

- 객체 지향 프로그래밍에서 사용하는 객체 개념을 기반으로 한 데이터 모델.
- 오라클 데이터베이스 -> 객체 관계형 DBMS로 영역 확장.



### 2.3. 관계형 데이터 모델

- 데이터 간 관계에 초점을 둠.
- 현대에 가장 많이 사용하는 모델.
- 각 데이터의 독립 특성만을 규정하여 데이터 묶음을 나누고, 중복이 발생할 수 있는 데이터는 별개의 릴레이션으로 정의한 후 데이터를 연결.
- 관계형 데이터 모델의 핵심 구성 요소
  - 개체(entity) : 데이터베이스에서 데이터화하려는 사물, 개념의 정보 단위. 관계형 데이터베이스의 테이블 개념과 대응되며 테이블은 릴레이션으로 표기하기도 함.
  - 속성(attribute) : 개체를 구성하는 데이터의 가장 작은 논리적 단위. 데이터의 종류, 특성, 상태 등을 정의함. 관계형 데이터베이스의 열 개념과 대응.
  - 관계(relationship) : 개체와 개체 또는 속성 간의 연관성을 나타내기 위해 사용. 관계형 데이터베이스에서는 테이블 간의 관례를 외래키(foreign key) 등으로 구현하여 사용함.
- 핵심 구성 요소를 활용하여 데이터의 독립성(independency)과 무결성(integrity)과 같은 데이터를 안전하게 관리하기 위해 필요한 개념들을 정의함.



## 3. 관계형 데이터베이스와 SQL



### 3.1. 관계형 데이터베이스란?

- 관계형 데이터 모델 개념을 바탕으로 데이터를 저장, 관리하는 데이터베이스.
- RDBMS : 관계형 데이터베이스를 관리하는 시스템.
- DBMS제품 대부분 관계형 데이터베이스 관리 시스템이거나 최소한 부분적으로 관계형 데이터베이스를 사용. 실무에서 쓰이는 DB의 대부분이 RDBMS 말함.
- DBMS제품 : MS-SQL(SQL-Server), MySQL(->MariaDB), PostgreSQL, DB2, 오라클 등



### 3.2. SQL이란?

- Structured Query Language.
- RDBMS 데이터를 다루고 관리하는 데 사용하는 데이터베이스 질의 언어.
- SQL 종류
  - DQL(Data Query Language) : RDBMS에 저장한 데이터를 원하는 방식으로 조회하는 명령어
  - DML(Data Manipulation Language) : RDBMS 내 테이블의 데이터를 저장, 수정, 삭제하는 명령어
  - DDL(Data Definition Language) : RDBMS 테이블을 포함한 여러 객체를 생성, 수정, 삭제하는 명령어
  - TCL(Transaction Control Language) : 트랜잭션 데이터의 영구 저장, 취소 등과 관련된 명령어. 제어문.
  - DCL(Data Control Language) : 데이터 사용 권한과 관련된 명령어

- 정의어(create, alter, drop 등) / 조작어(CRUD : insert into, select from where, update set, delete from 등) / 제어어(grant, revoke 등)
- ''시퀄''은 상표권 위반으로 현재의 SQL로 이름 바꿈.
