## 1. 변수

### 1.1. 변수의 정의

- 변수는 값을 저장할 수 있는 메모리의 특정 번지에 붙이는 이름을 말함.
- 변수를 통해 프로그램은 메모리 번지에 값을 저장하고 읽을 수 있음.
- 자바의 변수는 하나의 변수에 동시에 두 가지 값을 저장할 수 없고, 하나의 값만 저장할 수 있음.



### 1.2. 변수 선언

- 변수 선언은 변수에 어떤 타입의 데이터를 저장할 것인지 그리고 변수 이름이 무엇인지를 결정하는 것을 말함.
- 변수 이름은 자바 언어에서 정한 명명 규칙을 따름
  - 첫 번째 글자는 문자,'$','_' 이어야 하고 숫자로 시작할 수 없음.
  - 영어 대소문자를 구분
  - 첫 문자는 영어 소문자로 시작하되, 다른 단어가 붙을 경우 첫 문자를 대문자로 함 (관례)
  - 문자 수(길이)의 제한은 없음.
  - 자바 예약어는 사용할 수 없음.
  - 개발자는 변수가 어떤 값을 저장하고 있는지 쉽게 알 수 있도록 의미 있는 이름을 지어주어야 함.
- 예약어란 이미 해당 프로그래밍 언어에서 의미를 갖고 사용되고 있는 단어를 말함.
- 변수 초기화 : 변수에 최초로 값이 저장될 때 변수가 생성되는데 이것을 변수 초기화라고 함. 변수 초기화에 사용된 값을 초기값이라고 함.
- 초기화되지 않은 변수는 아직 메모리 번지 정보를 가지고 있지 않으므로 변수를 통해 메모리 값을 읽을 수 없음.



### 1.3. 변수 사용 범위

- 로컬 변수(local variable) : 메소드 블록 내에서 선언된 변수
- 로컬 변수는 메소드 실행이 끝나면 메모리에서 자동으로 없어짐.
- 변수는 블록 내 어디에서든 선언할 수 있지만, 선언된 블록 내에서만 사용할 수 있음.



## 2. 기본 타입

### 2.1. 정수 타입

- 리터럴(literal) : 소스 코드에서 프로그래머에 의해 직접 입력된 값. 입력된 리터럴 중에서 2진수, 8진수, 10진수, 16진수는 자바가 정수로 인식함.
- char 타입은 음수 값을 가질 수 없음.
- byte 타입 변수에 허용 범위를 초과한 값을 대입했을 경우 컴파일 에러가 발생함.
- long 타입은 수치가 큰 데이터를 다루는 프로그램에서 주로 사용됨. 대표적인 예는 은행이나 과학과 관련된 프로그램.
- 정수 리터럴이 int 타입의 허용 범위를 벗어날 경우 long 타입 정수 리터럴 뒤에 L 붙임. (약 20억 안쪽이면 안붙여도 상관없음.)



#### char 타입

- 문자 리터럴 : 하나의 문자를 작은따옴표(')로 감싼 것.
- 문자 리터럴은 유니코드로 변환되어 저장됨.
- char 타입도 정수 타입에 속해서 10진수 또는 16진수 형태로 유니코드를 저장할 수 있음.
- 작은따옴표로 감싼 문자 리터럴은 유니코드로 변환되기 때문에 int 타입 변수에도 저장할 수 있지만 출력하면 유니코드 자체가 출력됨.



### 2.2. String 타입

- 문자열 : 자바에서 큰따옴표(")로 감싼 문자들.
- 작은따옴표와 큰따옴표는 컴파일러가 문자 리터럴과 문자열 리터럴을 구별하는 기호로 사용됨.
- String 타입은 클래스 타입으로 기본 타입이 아님. 뒤에서 자세히 배울 예정.
- 이스케이프(escape) 문자 : 문자열 내부에 역슬래시(\)가 붙은 문자.
- 이스케이프 문자를 사용하면 문자열 내부에 특정 문자를 포함시킬 수 있음.
- 이스케이프 문자를 사용하면 출력을 제어할 수 있음. 탭만큼 띄우거나(\t) 개행(한 줄 내림, \n)을 지시할 수 있음.



### 2.3. 실수 타입

- 자바는 실수 리터럴을 기본적으로 double 타입으로 해석하기 때문에 double 타입 변수에 저장해야 함.
- 실수 리터럴을 float 타입으로 저장하고 싶다면 리터럴 뒤에 f나 F를 붙여 컴파일가 float 타입임을 알 수 있도록 해야 함.
- float 타입과 double 타입이 허용하는 소수점 이하 자리는 각각 7자리, 15자리임.



### 2.4. 논리(boolean) 타입

- 자바는 참과 거짓을 의미하는 논리 리터럴로 true와 false를 사용함.
- boolean 타입 변수는 주로 두 가지 상태값에 따라 조건문과 제어문의 실행 흐름을 변경하는 데 사용함.



## 3. 타입 변환

### 3.1. 자동 타입 변환

- 자동 타입 변환은 값의 허용 범위가 작은 타입이 허용 범위가 큰 타입으로 저장될 때 발생함.
- 허용 범위 크기순 : byte < short < int < long < float < double
- 정수 타입이 실수 타입으로 저장될 경우에는 무조건 자동 타입 변환이 일어남.
- char 타입이 int 타입으로 자동 타입 변환되면 유니코드 값이 int 타입에 저장됨.
- char 타입은 음수를 허용하지 않지만 byte 타입은 음수를 포함하기 때문에 byte 타입은 char 타입으로 자동 타입 변환되지 않음.



### 3.2. 강제 타입 변환

- 강제 타입 변환(Casting) : 큰 허용 범위 타입을 작은 허용 범위 타입으로 강제로 나눠서 저장하는 것.
- 강제 타입 변환은 캐스팅 연산자 괄호 ()를 사용하는데, 괄호 안에 들어가는 타입은 작은 허용 범위 타입임.
- int 타입을 char 타입으로 강제변환하면 문자를 출력할 수 있음.
- 실수 타입을 정수 타입으로 강제 타입 변환하면 소수점 이하 부분은 버려지고 정수 부분만 저장됨.



### 3.3. 정수 연산에서의 자동 타입 변환

- 정수 타입 변수가 산술 연산식에서 피연산자로 사용되면 int 타입보다 작은 byte, short 타입의 변수는 int 타입으로 자동 타입 변환되어 연산을 수행함.
- 정수 연산에서 사용되는 변수는 특별한 이유가 없는 경우 int 타입으로 선언하는 것이 타입 변환을 줄여주는 방법임. 타입 변환이 줄면 실행 성능이 향상됨.
- 직접적인 정수 리터럴을 연산하는 경우 피연산자가 변수가 아니므로 int 타입으로 변환을 하지 않음.
- 정수 연산에서 long 타입이 피연산자로 사용되면 연산 결과를 long 타입 변수에 저장해야 함.



### 3.4. 실수 연산에서의 자동 타입 변환

- 실수 타입 변수가 산술 연산식에서 피연산자로 사용될 경우 피연산자 중 하나가 double 타입이라면 다른 피연산자도 double 타입으로 자동 타입 변환되어 연산을 수행함. 연산 결과는 double 타입이 됨.
- 소문자 f 또는 대문자 F가 없는 실수 리터럴은 double 타입 변수에 저장함. float 타입 변수에 저장하면 컴파일 에러가 발생함.



### 3.5. '+' 연산에서의 문자열 자동 타입 변환

- '+' : 피연산자가 모두 숫자일 경우 덧셈 연산 수행, 피연산자 중 하나가 문자열일 경우에는 나머지 피연산자도 문자열로 자동 변환되어 문자열 결합 연산을 수행.
- 연산식에서 '+' 연산자가 연이어 나오면 앞에서부터 순차적으로 '+' 연산을 수행함. 먼저 수행된 연산이 덧셈 연산이라면 덧셈 결과를 가지고 다음 '+' 연산을 수행함. 먼제 수행된 연산이 결합 연산이라면 이후 '+' 연산은 모두 결합 연산이 됨.



### 3.6. 문자열을 기본 타입으로 강제 타입 변환

- String -> Int

  - ```java
    String str = "300000";
    int value = Integer.parseInt(str);
    ```

- String -> Double

  - ```java
    String str = "12.345";
    double value = Double.parseDouble(str);
    ```

- 문자열이 숫자가 아닌 알파벳이나 특수 문자, 한글 등을 포함하고 있을 경우 숫자 타입으로 변환을 시도하면 숫자 형식 예외(NumberFormatException)가 발생함.

- 기본 타입 -> String

  - ```java
    String str = String.valueOf(기본타입값);
    ```
